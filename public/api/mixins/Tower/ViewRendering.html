    Method Details

    ? (void) render(options, callback)

    The core method for rendering a view on the client or server.

    It is never called directly, but passed a more normalized options hash from the controller.

    Parameters:

    • options (Object)
    • callback (Function)

    Options Hash: (option):

    • type (String) — Can be json, coffee, ejs, html, etc. (i.e. anything in mint.js or with custom handlers defined). You can also skip this parameter and use it as a key.
    • json (Object) — Can pass json: {some: 'json'} (and any other "format")
    • text (String) — Raw text to write to the request.
    • template (String) — the path to the template relative to the app root (including file extension). If no template is given, it will try to compute the path from the type or default engine. This is the main (and most important) thing that gets computed in the render method, as it's the path to the actual file.
    • inline (Boolean) — whether or not the template is an inline function (rare)
    • layout (String|Boolean) — if false it won't render a layout, otherwise if a string it will grab the layout from app/layouts/<layout>.
    • locals (Object) — data accessible as variables in the template itself (the template context)
    • prefixes (Array) — Array of folders to look for the templates in.
